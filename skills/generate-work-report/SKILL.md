# Generate Work Report

## 适用场景

- 用户要做工作汇报、周报、月报、季度复盘、项目总结、方案汇报、述职 PPT
- 用户会明确指定 PPT 风格、排版、页数、字体、配色、页眉页脚、图表样式
- 用户只给零散要点、会议纪要、口头说明或半成品大纲，希望补全成可讲述的汇报稿

## 核心原则

- 先大纲，后 PPT
- 大纲必须可审、可改、可确认
- 未经用户确认，不进入 PPT 渲染
- 只有大纲通过后，才调用 `Presentations` 生成最终成稿
- 如果提供多个场景底板，先按场景选底板，再做内容填充和风格细化

## 输入约定

优先吸收这些信息：

- 汇报目标：给谁看、想推动什么结论
- 内容来源：原始要点、文档、表格、会议纪要、链接、图片、历史 PPT
- 风格约束：主题、配色、字体、版式、留白、封面、目录、页码、页脚、图表风格
- 交付约束：页数、比例、语言、是否需要演讲稿、是否需要备注区
- 底板约束：可选的场景模板、版式库、封面库、章节页库

当用户给出参考文档或参考 PPT 时，把它当成风格和结构约束来源；当只给风格偏好时，按用户定义的风格优先实现。
如果参考材料是链接但当前环境无法直接读取，就改用用户提供的截图、导出文件或粘贴文本继续，不要因为抓取失败中断生成流程。

## 工作原则

1. 先把原始内容整理成清晰的汇报主线，再输出大纲，不直接排版。
2. 大纲阶段允许适度扩写，但只扩写解释、过渡、总结、标题、注释和结构，不编造事实、数据、时间、结论。
3. 用户定义的风格和排版要求优先级高于默认审美。
4. 若内容不足以支撑某一页的结论，补充为“需要确认”而不是硬编数字。
5. 最终输出必须是可编辑的 PPTX，不是只有图片的静态稿。
6. 大纲确认前，不生成最终 PPT，也不做最终视觉渲染。

## 生成流程

### 阶段一：生成大纲

1. 识别汇报类型和受众。
2. 提取必须保留的事实、数字、结论和禁区。
3. 抽取风格规则，整理成可执行的设计约束。
4. 重构内容为页面级大纲。
5. 为每页写出：
   - 页序
   - 页标题
   - 页面目的
   - 主要内容点
   - 需要补充/待确认的信息
   - 建议的图表或版式类型
   - 推荐使用的场景底板
6. 明确标记这是“待确认大纲”，并请求用户确认后再进入下一阶段。

大纲输出建议使用结构化文本，必要时可附带简短表格，便于人工审阅和修改。

### 阶段二：渲染 PPT

只有在用户确认大纲后，才进入这一阶段。

1. 根据已确认大纲补齐页面结构和视觉节奏。
2. 如果存在多个底板候选，先选择最贴合场景的底板，再在此基础上生成最终稿。
3. 把碎片化要点重组为“背景 -> 进展 -> 结果 -> 问题 -> 动作项”或更贴合场景的结构。
4. 为每一页写出结论型标题，而不是主题型标题。
5. 为每页补充必要的解释句、承接句、过渡句和行动建议。
6. 使用 `Presentations` 技能作为底层生成器。
7. 按页创建可编辑元素：标题、正文、表格、图表、时间线、流程图、对比块、结论块。
8. 所有重要结论都要有视觉支撑，不要堆纯文本。

### 3. 锁定视觉系统

- 固定封面、章节页、内容页、总结页、附录页的统一规则
- 将用户指定的颜色、字体、间距、对齐方式、图表样式、页脚规范固化为全局规则
- 若用户未指定，则使用克制、清晰、适合汇报场景的默认系统

### 4. 渲染和校正

- 预览缩略图，检查封面、目录、章节切分、层级、对齐、留白、信息密度
- 优先修正最弱的页面，而不是平均分配精力
- 检查是否严格遵守用户定义的风格和版式规则

## 页面组织建议

按内容选择最合适的结构，不强行套模板：

- 周报 / 月报：本期概览、关键进展、问题与风险、下期计划
- 项目汇报：背景目标、执行过程、阶段成果、问题阻塞、下一步动作
- 方案汇报：现状分析、方案对比、推荐方案、实施路径、资源诉求
- 复盘总结：目标回顾、实际结果、原因拆解、经验沉淀、后续优化

## 输出要求

- 阶段一输出：大纲草案，等待用户确认
- 阶段二输出：最终 `.pptx`
- 如用户需要，可附带一份简短的页面大纲或讲稿提纲
- 保持文件命名清晰、主题明确
- 所有中间产物遵循 `Presentations` 技能的工作区规则

## 底板放置建议

把场景底板统一放在本 skill 的 `assets/templates/` 下，例如：

- `assets/templates/weekly-report.pptx`
- `assets/templates/monthly-report.pptx`
- `assets/templates/project-review.pptx`
- `assets/templates/proposal.pptx`

如果后续底板很多，再补一个 `assets/templates/index.json`，记录：

- `scene`
- `file`
- `适用场景`
- `页型特征`
- `默认主题色`
- `是否适合封面复用`
