# 第 5 步报告样例

这是第 5 步确认版报告的样例页，用来约束人工确认后的代码变更写法和章节顺序。这里的内容是结构示例，不是实际 case 结论。

## 1. 人工确认后的变更项

| 优化点 | 确认结果 | 修改范围 | 说明 |
|---|---|---|---|
| 历史扫描 | 确认可改 | `foo.py:10-40` | 收窄窗口并裁字段 |
| 冗余字段 | 确认可删 | `foo.py:41-60` | 删除未消费字段 |

## 2. 优化前 / 优化后代码快照对比

- 优化前代码：`source_before/`
- 优化后代码：`source_after/`
- 变更摘要：保留最小必要改动

## 3. 源码 -> 代码变更映射图

- 图里必须能看见原始瓶颈位置
- 图里必须能看见人工确认后的修改点
- 图里必须能看见修改后代码草图

## 4. 未解决的问题

- 如果某些点不能精确落到代码，只写修改思路，不硬填代码
- 如果某些点暂时不能改，要说明原因
